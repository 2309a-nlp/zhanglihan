## 二、严重问题（建议立即修复）

### 1. `create_agent` 函数不存在

**位置**：`app.py` 第 17 行
```python
from langchain.agents import create_agent        # 这个函数不存在
```

**问题**：`langchain.agents` 模块中**没有** `create_agent` 函数。在 LangChain 1.x 中应使用 `initialize_agent`，在 LangChain 2.x / LangGraph 中应使用 `create_react_agent`。

**影响**：当前虽测试通过，但依赖的是运行环境中某个兼容版本的意外兜底行为，升级 langchain 后必定崩溃。

**建议修复**：
```python
from langgraph.prebuilt import create_react_agent
```

---

### 2. 废弃的 API 参数名

**位置**：`app.py` ChatOpenAI 初始化
```python
llm = ChatOpenAI(
    openai_api_key=DEEPSEEK_API_KEY,    # 废弃参数
    openai_api_base=DEEPSEEK_BASE_URL,  # 废弃参数
)
```

**问题**：`openai_api_key` 和 `openai_api_base` 在 `langchain-openai >= 0.2.0` 中已废弃，应改为 `api_key` 和 `base_url`。当前虽能工作但会触发 DeprecationWarning，未来版本会删除。

**建议修复**：
```python
llm = ChatOpenAI(
    model="deepseek-chat",
    api_key=DEEPSEEK_API_KEY,
    base_url=DEEPSEEK_BASE_URL,
)
```

### 3. 数据库连接没有使用上下文管理器

**位置**：整个 `app.py` 中多处

**问题**：所有 `get_db_connection()` 都是手动 `conn.close()`，如果中间抛出异常会导致连接泄露。

**当前代码（重复 7 次）**：
```python
conn = get_db_connection()
cursor = conn.cursor()
cursor.execute(...)
conn.commit()
conn.close()     # 如果前面异常，这里不会执行
```

**建议修复**：使用 `with` 语句或上下文管理器：
```python
from contextlib import contextmanager

@contextmanager
def get_db():
    conn = sqlite3.connect(DB_PATH)
    conn.row_factory = sqlite3.Row
    try:
        yield conn
    finally:
        conn.close()
```

## 三、代码质量问题

### 4. 提醒检查逻辑严重重复

检查到提醒的 SQL 查询在代码中出现了 **3 次**：

| 位置 | 函数 |
|------|------|
| `app.py` | `get_pending_reminders()` 工具函数 |
| `app.py` | `check_reminders()` Flask 路由 |
| `app.py` | `check_and_get_reminders()` 内部函数 |

**问题**：相同的查询逻辑复制粘贴了 3 次，修改一处忘改其他两处会导致 bug。

**建议**：抽取为一个共享函数：
```python
def _query_pending_reminders(conn) -> list:
    today = datetime.now().strftime("%Y-%m-%d")
    now_time = datetime.now().strftime("%H:%M")
    conn.execute(...)
    return conn.fetchall()
```

### 5. 生产模式使用 `debug=True`

**位置**：`app.py` 最后一行
```python
app.run(debug=True, host="127.0.0.1", port=5000)
```

**问题**：
- debug 模式会加载调试器，有安全风险
- 热重载会重启两次服务器，影响性能

**建议**：
```python
debug = os.getenv("FLASK_DEBUG", "0") == "1"
app.run(debug=debug, host="127.0.0.1", port=5000)
```

---

### 6. 使用 `print` 而非 `logging`

**问题**：全项目使用 `print()` 输出日志，无法控制日志级别，无法输出到文件。

**建议**：
```python
import logging
logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(message)s",
    handlers=[logging.StreamHandler(), logging.FileHandler("app.log", encoding="utf-8")]
)
logger = logging.getLogger(__name__)
```

### 7. 系统提示词过长（约 130 行）

**问题**：系统提示词约 2000+ tokens，包含大量示例和规则。每次 Agent 调用都会发送，消耗大量 token，增加响应延迟。

**建议**：
- 将"口语化理解"部分用更简洁的规则表达式替代
- "对话风格"压缩为 2-3 句话

## 五、部署问题

### 11. 依赖版本未锁定

**位置**：`requirements.txt`
```
flask>=3.0.0
langchain>=0.3.0

```

**问题**：使用 `>=` 范围，不同时间安装可能得到不同版本，行为不一致。且缺少 `python-dotenv` 依赖声明。

**建议**：
```

langchain==1.3.2
langchain-openai==1.2.1
python-dotenv==1.2.2
```
