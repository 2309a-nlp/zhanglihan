# Stable Diffusion WebUI 部署设计文档

## 1. 概述

### 1.1 项目信息

| 项目 | 值 |
|------|------|
| 应用名称 | AUTOMATIC1111 Stable Diffusion WebUI |
| 部署版本 | v1.10.1 (commit 82a973c0) |
| 部署路径 | E:\sd-webui |
| 操作系统 | Windows 11 |
| 部署日期 | 2026 年 6 月下旬 |
| 状态 | 运行中（已生成测试图像） |

### 1.2 功能概述

基于 AUTOMATIC1111 的 Stable Diffusion WebUI，提供以下核心功能：

- **文生图 (txt2img)**：通过文本提示词生成图像
- **图生图 (img2img)**：基于参考图像进行风格迁移或修改
- **图像修复 (inpaint)**：局部区域重绘
- **高清修复 (hires fix)**：低分辨率生成 + 超分放大
- **ControlNet 姿态控制**：使用 OpenPose 预处理器进行人体姿态引导生成
- **模型融合 (model merger)**：多模型权重混合
- **图像后处理 (extras)**：超分放大、面部修复、图像裁剪等
- **XYZ 网格 (xyz grid)**：批量参数对比实验
- **LoRA 加载**：支持轻量级微调模型加载


## 8. 已知的坑与修复记录

### 8.1 网络受限环境适配

| 问题 | 修复 |
|------|------|
| GitHub 被墙 | Git remote 已修正为官方地址，未使用 gitclone.com 代理 |

### 8.2 依赖兼容性问题

| 问题 | 修复 |
|------|------|
| numpy >= 2.0 导致 `numpy.dtype size changed` | 锁定 numpy 1.26.2 |
| mediapipe >= 0.10.9 导致 `mp.solutions` 不存在 | 降级至 mediapipe 0.10.7 |
| insightface 安装时 GitHub wheel 被墙 | 通过清华 pip 镜像安装 |
| 每次 pip install 可能重新拉高 numpy | 安装后必须检查并重新降级 numpy |

### 8.3 ControlNet 安装问题

| 问题 | 修复 |
|------|------|
| ZIP 安装导致嵌套目录 | 手动将文件上移一级，确保 `scripts/` 在扩展根目录下 |
| 启动时卡住无输出 | 添加 `--skip-version-check` 和 `--skip-prepare-environment` |
| 预处理器模型下载被墙 | 需手动从 hf-mirror 下载到 `annotator/downloads/` |

## 11. 维护建议

### 11.1 日常维护

1. **定期检查 numpy 版本**：任何 `pip install` 后执行 `python -m pip show numpy` 确认版本 < 2.0
2. **模型管理**：新增模型放入对应 `models/` 子目录
3. **输出清理**：定期清理 `outputs/` 目录中的旧图像

### 11.2 扩展建议

- **下载 ControlNet 预处理器模型**：首次使用 OpenPose 前需下载 5 个预处理器模型（约 580 MB）
- **添加更多 ControlNet 模型**：Canny、Depth、MLSD 等可根据需求下载
- **添加 VAE 模型**：当前使用模型内置 VAE，可单独下载高质量 VAE 改善色彩
