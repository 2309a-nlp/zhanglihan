## 二、优化项明细

### 2.1 检索参数优化（app.py）

| 参数 | 优化前 | 优化后 | 效果 |
|------|-------|-------|------|
| `top_k` | 10 | **3** | 文档数减少 70%，LLM 输入 tokens 从 8000+ 降至 ~1500 |
| 默认重排器 | `"llm"` | **`"tfidf"`** | 不再加载 2.1GB BGE-Reranker，启动快 ~45 秒 |

### 2.2 LLM 参数优化（qa_engine.py, config.py）

| 参数 | 优化前 | 优化后 | 效果 |
|------|-------|-------|------|
| `max_tokens` | 2048 | **512** | 回复更简洁，生成速度提升 |
| `LLM_TIMEOUT` | 40 秒 | **30 秒** | 失败快速降级 |
| `call_llm timeout` | 40 秒 | **30 秒** | 失败快速降级 |
| 文档截断 | 800 字符 | **500 字符** | 输入更精简 |

### 2.3 重排器自动检测本地模型（rerankers.py）

在 `LLMReranker.__init__` 中新增自动检测本地路径逻辑：

```python
local_path = os.path.join(
    os.path.dirname(__file__), 
    "models", model_name.split("/")[-1]
)
if os.path.exists(local_path):
    self.model_path = local_path
```

**效果：** 默认不加载 LLM 重排器；用户若切换为 `"llm"` 重排方式，自动使用本地 `models/bge-reranker-v2-m3` 模型，不再从 HuggingFace 在线下载。

### 2.4 嵌入模型预热（app.py）

```python
# initialize_rag() 中新增
if ok:
    embed_query("warmup", model_name=st.session_state.embedding_model)
```

**效果：** 初始化阶段完成 SentenceTransformer 模型加载，首次查询不再有 ~11 秒的模型加载延迟。

### 2.5 混合检索归一化公式修复（hybrid_search.py）

```python
# 修复前（导致分数分布不合理）
normalized = (score - (v_max - v_range)) / v_range

# 修复后
normalized = (score - v_min) / (v_max - v_min) if v_max != v_min else 0.0
```

**效果：** 向量检索分和全文检索分正确归一化到 [0, 1]，混合权重 `vector_weight=0.6` 按预期生效。
