# RAG 知识库问答系统 — 设计文档

## 1. 系统概述

本系统是一个轻量级、离线可用的 RAG（Retrieval-Augmented Generation）问答系统，专门用于招股说明书的智能问答。

### 核心设计原则

1. **离线可用**：使用 TF-IDF 检索，无需下载 Embedding 模型或连接外部 API
2. **快速响应**：结构化数据采用代码级提取，绕过 LLM 推理延迟
3. **精确回答**：财务表格数据通过正则提取，避免小模型幻觉
4. **渐进增强**：结构化查询 < 0.1s，复杂查询回退到 LLM

## 3. 核心流程

### 3.1 知识库构建

```
PDF文件 → PyMuPDF提取文本 → RecursiveCharacterTextSplitter分块
  → jieba分词 → TfidfVectorizer(char_wb, 2-6-gram) → 缓存到.tfidf_cache/
```

- **分块策略**：chunk_size=512, chunk_overlap=64
- **分隔符优先级**：`\n\n` > `\n` > `。` > `；` > `，` > ` `
- **向量化参数**：analyzer='char_wb', ngram_range=(2,6), max_features=50000

### 3.2 查询流程

```
用户问题
  │
  ├─→ jieba分词 + 关键词扩展 + 类别标记
  │    (军用→军用 国防 军队 军方 军品 军工)
  │    (领域→领域 范围 方向 板块 方面 部门)
  │    (收入→收入 营收 销售额 金额)
  │
  ├─→ TF-IDF向量化 + 余弦相似度 → Top-8 Chunks
  │
  ├─→ 表数据提取（正则匹配国防/军用/直接军方等模式）
  │    ├── 命中 → 构建Markdown表格回答 (0.05s)
  │    └── 未命中 → 构造LLM Prompt → Ollama推理 → 回答 (<6s)
  │
  └─→ 返回结构化回答 + 来源信息
```

### 3.3 多轮对话

- 对话历史存储在 `_conversation_history` 列表（最多20轮）
- 追问通过 `_best_question()` 合成完整问题
- 历史信息仅用于 LLM 回退路径，结构化查询直接使用当前问题关键词

## 4. 关键技术决策

### 4.1 为什么用 TF-IDF 而不是 Embedding？

- **离线可用**：无需下载 Embedding 模型或连接外部 API
- **快速索引**：300页PDF索引构建 < 5秒
- **轻量级**：内存占用 < 200MB
- **适用于中文金融文档**：char_wb n-gram 对表格、数字、短文本检索效果好

### 4.2 为什么用正则提取而非 LLM 解析表格？

测试表明 1.5B 参数模型（qwen2.5:1.5b）对中文跨行表格的解析准确率不足 60%。采取「代码精确提取 + LLM格式化」的混合策略：

| 方案 | 准确率 | 延迟 |
|------|--------|------|
| 纯 LLM 解析 | ~60% | ~10s |
| 代码提取 | 100% | <0.01s |
| 混合策略 | 100% | <0.1s |

### 4.3 查询扩展机制

```
用户输入: "军用领域收入"
  │
  ├─ jieba分词: ["军用", "领域", "收入"]
  │
  ├─ 类别标记:
  │   military: [军用, 国防, 军方]
  │   domain:   [领域, 范围, 方向]
  │   revenue:  [收入, 营收]
  │
  └─ 组合查询: "军用 国防 领域 收入" "军方 收入 营收" ...
```

## 6. 性能优化

| 优化项 | 方法 | 效果 |
|--------|------|------|
| 索引缓存 | 持久化 TfidfVectorizer + 稀疏矩阵 | 二次加载 < 1s |
| 合并查询 | 多组扩展词合并为单次 TF-IDF 变换 | 减少 7 倍计算 |
| 表提取先行 | 正则匹配优先于 LLM 调用 | 响应 < 0.1s |
| LLM 限制 | num_ctx=3072, num_predict=768 | 减少生成延迟 |

## 7. 局限与改进方向

1. **LLM 回退路径延迟**：qwen2.5:1.5b 在非结构化查询上需 ~5-6s，可升级模型或改用 API
2. **表格覆盖度**：当前仅支持固定模式的财务表格，可扩展正则覆盖更多类型
3. **并发支持**：Streamlit 原生单线程，可考虑 FastAPI + 异步
4. **跨文档查询**：当前按 chunk score 合并，可增加交叉文档推理
