# 工单05 项目优化报告

## 项目概述

工单05 是一个基于 **TF-IDF + Multi-Query** 的 RAG 问答系统，针对武汉兴图新科招股说明书提供结构化财务数据问答。

- **检索方式**: TF-IDF (char_wb, 2-6 ngram, 50000 特征)
- **查询扩展**: Multi-Query (2-3 个变体查询)
- **LLM**: DeepSeek Chat (urllib)
- **UI**: Streamlit
- **索引规模**: 2176 个 chunk，6.2MB 矩阵

## 2. 优化方案

### 2.1 语义缓存（最大收益，预估 -50% LLM 调用）

当前每次相同问题都要调 API。增加 LRU 语义缓存：

```python
# 新增 _semantic_cache = {}
# key = f"{question}:{top_k}"
# 命中直接返回缓存的 answer

_SEMANTIC_CACHE = {}  # LRU dict
_CACHE_MAX = 50

def _get_cache_key(question):
    # 用 extract_keywords + question 做 key
    kws = " ".join(extract_keywords(question, max_terms=6))
    return f"{kws}|{question}"

def query(question):
    cache_key = _get_cache_key(question)
    if cache_key in _SEMANTIC_CACHE:
        return _SEMANTIC_CACHE[cache_key]
    result = _do_query(question)
    if len(_SEMANTIC_CACHE) >= _CACHE_MAX:
        _SEMANTIC_CACHE.pop(next(iter(_SEMANTIC_CACHE)))
    _SEMANTIC_CACHE[cache_key] = result
    return result
```

- **预估效果**: 重复问题秒回（0.1s vs 2s），常见问题命中率 >60%

### 2.2 预热机制（启动时预检索）

当前首次查询包含 ~5s 的模块导入 + 索引加载。增加 warmup：

```python
def warmup():
    """启动预检，避免首请求卡顿"""
    _ = retrieve("预热", k=2)
    _ = extract_keywords("预热查询", max_terms=4)

# 在 init_rag() 最后调用 warmup()
```

### 2.3 连接池复用（LLM API 调用优化）

当前每次 query 新建 urllib Request。复用连接可节省 DNS + SSL 握手：

```python
import http.client

_LLM_CONN = None

def _get_llm_conn():
    global _LLM_CONN
    if _LLM_CONN is None:
        _LLM_CONN = http.client.HTTPSConnection("api.deepseek.com", timeout=LLM_TIMEOUT)
    return _LLM_CONN
```

### 2.4 熔断器（防连续失败）

API 连续失败时自动降级，避免浪费等待：

```python
_CIRCUIT_BREAKER = {"failures": 0, "last_fail": 0, "open": False}
_CIRCUIT_THRESHOLD = 3
_CIRCUIT_RESET = 60  # 60秒后自动半开

def _check_circuit():
    cb = _CIRCUIT_BREAKER
    if cb["open"]:
        if time.time() - cb["last_fail"] > _CIRCUIT_RESET:
            cb["open"] = False  # 半开
            return True
        return False
    return True

def _record_failure():
    cb = _CIRCUIT_BREAKER
    cb["failures"] += 1
    cb["last_fail"] = time.time()
    if cb["failures"] >= _CIRCUIT_THRESHOLD:
        cb["open"] = True

def _record_success():
    _CIRCUIT_BREAKER["failures"] = 0
```

### 2.5 流式输出（用户体验优化）

当前等待全部生成完才显示。改为 SSE 流式：

```python
def stream_query(question):
    """流式版本 query，逐 token 返回"""
    # ... same retrieval ...

    req = urllib.request.Request(url, data=json.dumps(payload).encode(), headers=headers)
    with urllib.request.urlopen(req, timeout=LLM_TIMEOUT) as resp:
        for line in resp:
            line = line.decode().strip()
            if line.startswith("data: ") and line != "data: [DONE]":
                chunk = json.loads(line[6:])
```

## 5. 已知问题

1. **PYTHONPATH 冲突**: 系统 Python312 和 conda zg5 环境的 site-packages 互相干扰，启动时需 `set PYTHONPATH=`
2. **pip SSL 故障**: `_ssl.c:997 EOF` 导致无法通过 pip 安装包，需手动复制
3. **Extract_keywords 中 jieba 精度**: 对财务专业术语的分词效果一般，可考虑添加自定义词典

---

## 6. 建议优先级

1. ⭐ 语义缓存（实现简单，收益最大）
2. ⭐ 预热机制（两行代码，消除冷启动）
3. 连接池复用（减少 0.3-0.5s）
4. 熔断器（稳定性保障）
5. 流式输出（体验优化，需配合 Streamlit 改动）
