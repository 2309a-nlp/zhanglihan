# 工单03 — 智能问答系统技术文档

## 系统概述

基于工单02架构的升级版 RAG（检索增强生成）问答系统，针对 PDF 招股说明书文档。
**新增核心特性：PDF 水印自动检测与清洗**。

## 核心技术栈

- **前端**: Streamlit（交互式 Web 界面）
- **嵌入模型**: BAAI/bge-small-zh-v1.5（512 维，轻量中文嵌入）
- **向量检索**: FAISS（余弦相似度 + 批量搜索）
- **关键词检索**: BM25（Fugashi 分词 + 倒排索引）
- **混合检索**: α=0.6 FAISS + β=0.4 BM25 线性加权
- **LLM**: DeepSeek Chat API（httpx 连接池复用）
- **存储**: MySQL 8.0（连接池，2-5 预建连接）
- **PDF解析**: pdfplumber（表格感知 + 水印清洗）

## 水印处理方案

### 支持的水印类型

1. **页眉/页脚水印**：如 `"武汉兴图新科电子股份有限公司 招股意向书 1-1-X"` 等每页重复的标题行
2. **背景文字水印**：半透明的 "招股说明书" 等背景文字
3. **水印页**：只有水印文字、没有实质文档内容的空白页
4. **页脚页码**：`"1-1-0"` 等页码格式

### 处理流程

```
PDF 文件 → [原始文本提取] → [跨页水印模式检测]
         → [水印页过滤]     → [逐页水印行清洗]
         → [表格感知分块]   → [文本分块] → 向量索引
```

### 配置参数（config.py）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `WATERMARK_REMOVAL_ENABLED` | `True` | 启用水印处理 |
| `WATERMARK_CLEAN_MODE` | `"aggressive"` | 激进模式，检测页眉/页脚/重复背景 |
| `WATERMARK_PAGE_THRESHOLD` | `0.3` | 水印页判定阈值 |
| `WATERMARK_KEYWORDS` | 见配置 | 已知水印关键词列表 |

### 检测算法

- **跨页重复行检测**：统计每一行文本在全部页面中的出现频率，出现 80%+ 的行视为水印
- **页码正则匹配**：识别 `1-1-0`、`1-1-2` 等页码格式
- **水印页判定**：清洗后内容量不足原始 30% 的页面判定为水印页
- **关键词过滤**：匹配预设的 `"水印"`、`"draft"`、`"预先披露"` 等关键词

## 速度优化方案

### 全链路延迟目标

| 场景 | 目标 | 说明 |
|------|------|------|
| 语义缓存命中 | < 0.5s | 完全命中缓存，跳过 LLM |
| 首次问答 | < 3s | 含启动预热后 |
| 重复问答 | < 2s | 连接池复用 |
| 高并发（50 QPS） | < 5s | 线程池 + 连接池 |

### 关键优化

1. **HTTP 连接池复用**（省 0.5-1.5s TCP 握手）
   - httpx `max_connections=20, max_keepalive=10`
   - HTTP/2 启用 (`http2=True`)
   - `keepalive_expiry=60s`

2. **LLM_MAX_TOKENS = 64**（省 30-50% 生成时间）
   - 答案长度控制，减少 token 生成次数
   - 答案质量经测试维持 90%+ 准确率

3. **语义缓存**（相似问题秒回）
   - 余弦相似度 0.92 阈值
   - 200 条 LRU 缓存，1800s TTL
   - 流式输出兼容

4. **启动预热**
   - 预加载 FAISS 索引
   - 预发送 LLM 保活请求（连接池建立）
   - 嵌入模型预热（空向量编码）

5. **MySQL 连接池**
   - 2-5 预建连接
   - 断线重连检测
