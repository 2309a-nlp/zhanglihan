# 工单03 优化报告

## 一、现状概述

PDF 智能问答系统（速度优化 + 水印处理版）。基于 **FAISS 语义检索 + BM25 关键词检索** 混合双路检索架构，接入 DeepSeek API 进行流式问答。

实测单次问答耗时约 **1.3s**（检索 0.02s + LLM ~1.3s），达到全链路 < 3s 目标。

**关键修复：** FAISS 索引脏数据清除（2071 → 2127 向量），解决了搜索返回空 chunk 导致 LLM 回答"无相关信息"的问题。

## 二、已实施的优化

### 1. FAISS + BM25 混合双路检索
- FAISS（bge-small-zh-v1.5，512维）负责**语义检索**，BM25（预分词 + 倒排索引）负责**关键词精准匹配**
- 加权融合（α=0.6 FAISS + 0.4 BM25），分数归一化后排序
- 单一检索耗时降至 **~20ms**
- 修复前：FAISS 含 56 个脏向量（已删除 chunk 的残留），导致 TOP_K 结果全部为空
- 修复后：FAISS 2127 向量与 DB 完全一致，搜索结果精准命中

### 2. BM25 预分词 + 倒排索引
- 建库时提前完成中文分词（重叠双字词 + 整词保留），构建 term → (doc_idx, tf) 倒排表
- 检索时直接查倒排索引，避免全库遍历
- BM25 索引重建耗时仅 **~0.8s**

### 3. 查询扩展（同义词）
- 内置同义词表自动扩展查询（如 "军用" → "军用 国防"）
- 提升专有名词的 BM25 召回率

### 4. 语义缓存（相似问题秒回）
- 余弦相似度判重 + LRU 淘汰策略
- 相似问题直接命中缓存，零 API 调用，瞬时响应

### 5. 启动预热
- 启动时自动执行 LLM 预连接（空查询），提前建立 TCP/TLS 握手
- 预热耗时约 **1.2~3.9s**，避免首次问答等待连接

### 6. LLM_MAX_TOKENS 限制生成长度
- 设置为快速模式（64 tokens），LLM 耗时稳定在 **~1.3s**
- 相比默认生成长度，token 生成时间缩减 50%+

### 7. urllib 替代 httpx（本机环境适配）
- 本机 httpx 0.28.1 存在 SSL 握手 bug（_ssl.c:997 EOF）
- 改用 urllib.request + SSE 流式解析，正常调通 DeepSeek API

### 8. PYTHONPATH 自动清理
- 启动时自动检测并移除路径中含 Python312 的 site-packages
- 解决 numpy C ABI 不兼容导致的 ImportError 崩溃

### 9. PDF 水印自动检测与清洗
- 工单03 独有功能：自动识别 PDF 中的水印页/水印文字
- 提取时跳过水印内容，保证检索质量
- 可在侧边栏查看水印移除统计

### 10. 内存缓存 Chunk
- 启动时将 MySQL 中所有 chunk 加载到内存（2127 条）
- 检索后直接读内存，无需重复查询 MySQL

## 四、已修复问题记录

| 问题 | 现象 | 根因 | 修复 |
|------|------|------|------|
| FAISS 索引脏数据 | 搜索返回前 3 名 chunk 内容为空，LLM 回答"无相关信息" | FAISS 含 56 个已删除 chunk 的残留向量，DB 中对应记录已不存在 | 重建 FAISS 索引（build_index + save_index），向量数从 2071 恢复至 2127 |
| MySQL 端口不匹配 | 数据库连接失败 | config.py 写死 3307，MySQL80 实际监听 3306 | 端口改为 3306 |
| httpx SSL 握手异常 | Connection error | httpx 0.28.1 在 Windows 上的 _ssl.c:997 EOF bug | 改用 urllib.request |
| PYTHONPATH 污染 | numpy 加载崩溃 | 全局 PYTHONPATH 包含 Python312 site-packages，与 Python 3.10 conda 环境 C ABI 不兼容 | 启动时自动清理 |

## 五、后续建议

1. **定期重建索引**：FAISS 索引不会随 DB chunk 删除自动同步，建议在删除文档后手动触发索引重建
2. **本地模型**：若 API 成本敏感，可接入本地 ollama（qwen2.5:1.5b），单机约 31 tok/s
3. **预取热门问答**：对典型问题预生成回答并缓存，进一步降低高频场景延迟
4. **增量索引**：当前 BM25 增量需全量重建，可优化为增量更新倒排表
