# 工单02 优化报告

## 一、现状概述

PDF 智能问答系统，基于 BM25 检索 + DeepSeek API。
实测单次问答耗时约 **1.4s**（检索 0.03s + LLM 1.4s）。

## 二、已实施的优化

### 1. BM25 预分词 + 倒排索引
- 建库时提前完成分词，检索时直接查倒排索引
- 检索时间降至 **~30ms**

### 2. TF-IDF（本地检索，零模型下载）
- 采用 sklearn TfidfVectorizer(char_wb, 2-6 n-grams) + jieba
- 无需下载任何 embedding 模型，适合本机网络环境

### 3. 语义缓存
- 余弦相似度 + LRU 淘汰
- 相似问题直接命中缓存，秒回，零 API 调用

### 4. 启动预热
- 启动时自动发一次空查询，建立网络连接
- 避免第一次用户提问等待 TCP/TLS 握手

### 5. LLM_MAX_TOKENS=64 限制最大生成长度
- LLM 耗时从 3-5s 压缩至 ~1.4s

### 6. urllib 替代 httpx（本机适配）
- 本机 httpx 0.28.1 SSL 握手存在 bug
- 改用 urllib.request，正常调通 DeepSeek API

### 7. PYTHONPATH 自动清理
- 启动时自动移除不兼容的 Python312 site-packages

## 三、当前瓶颈

| 环节 | 耗时 | 占比 |
|------|------|------|
| BM25 检索 | ~0.03s | 2% |
| DeepSeek API | ~1.4s | 98% |

**主要瓶颈在 DeepSeek API 响应速度**，本地优化空间有限。

## 四、后续建议

1. **改用更快模型**（DeepSeek fast 模型或本地 ollama qwen2.5:1.5b）
2. **预取/预缓存**：对常见问题预生成回答
3. **精简文档上下文**：减少输入 token 数可加快首 token 生成
