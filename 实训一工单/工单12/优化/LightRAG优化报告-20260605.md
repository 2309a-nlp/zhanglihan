## 二、部署中的问题与优化建议

### 问题 1：uv tool 隔离环境依赖不全

**现象：**  
`uv tool install lightrag-hku` 安装后，启动 `lightrag-server` 时报多个 `ModuleNotFoundError`。

| 缺失模块 | 原因 |
|---------|------|
| `fastapi` / `uvicorn` | lightrag-hku 未标注为正式依赖 |
| `ollama` | Embedding binding 的 Optional 依赖 |
| `pyjwt` / `bcrypt` / `passlib` | HTTP API 认证依赖 |
| `aiofiles` / `aiosqlite` | 异步文件/数据库操作 |

**优化建议：**
- ✅ **短期方案（已实施）：** 在 uv 环境内 `python -m ensurepip` + `pip install` 逐个补齐。
- ⚡ **长期方案：** 创建独立的 conda 虚拟环境，用 `pip install lightrag-hku[all]` 一次性安装所有依赖，避免 uv 隔离层带来的依赖发现问题。
- 🔧 或改用 `uv tool install lightrag-hku --with fastapi,uvicorn,ollama,pyjwt,bcrypt,...` 在安装时就带上全部依赖。

### 问题 2：启动命令过长、配置分散

**现象：**  
启动命令需要同时传多个 `--*` 参数、环境变量存在 `.lightrag_env` 文件、且每次在不同终端执行时容易遗漏参数。

```bash
# 当前启动方式 —— 参数多且分散
export OPENAI_API_KEY="sk-..."
export OPENAI_API_BASE="https://api.deepseek.com/v1"
lightrag-server \
  --llm-binding openai \
  --llm-model deepseek-chat \
  --embedding-binding ollama \
  --ollama-host http://172.30.208.1:11434 \
  --embedding-model qwen2.5:1.5b
```

**优化建议：**
- ✅ 创建 `start_lightrag.sh` 启动脚本，固化所有参数和环境变量。
- ✅ 将 `.lightrag_env` 用 `source` 方式加载（而不是手动 export）。
- ⚡ 配置 LightRAG 的 YAML 配置文件（若支持），将 LLM/Embedding 绑定写入配置文件而非命令行参数。

### 问题 3：Windows Ollama 嵌入支持未启用

**现象（阻塞性）：**  
`/api/embed` 端点返回错误 `"This server does not support embeddings"`，因为 Windows 版 Ollama 默认**不开启**嵌入功能。

**优化建议：**
- ✅ **必须操作：** 关闭当前 `ollama serve` → 打开**新的 CMD** → 执行 `set OLLAMA_EMBEDDINGS=1` → 再启动 `ollama serve`（注意：必须在新 CMD 窗口，不是 PowerShell）。
- ⚡ **自动化：** 创建 Windows 批处理文件 `ollama-with-embed.bat`，内容：
  ```bat
  @echo off
  set OLLAMA_EMBEDDINGS=1
  ollama serve
  ```
  以后双击此文件启动 Ollama 即可自动开启嵌入。
- 🔧 **后备方案：** 如果 Ollama 嵌入始终不可用，可改用 DeepSeek 的嵌入 API（`text-embedding-ada-002` 兼容）或本地 SBERT 模型（base conda 环境已有 `sentence-transformers`）。

### 问题 4：WSL 网关 IP 不固定

**现象：**  
Ollama 通过 WSL 网关 IP `172.30.208.1` 访问 Windows，但该 IP 在每次 WSL 重启后可能变化。

**优化建议：**
- ✅ 使用 `ip route show default` 动态获取网关 IP，写入启动脚本：
  ```bash
  GATEWAY_IP=$(ip route show default | awk '{print $3}')
  export OLLAMA_HOST="http://${GATEWAY_IP}:11434"
  ```
- ⚡ 或将 Windows Ollama 端口映射到 WSL 内固定地址（如 `127.0.0.1:11434`），通过 Windows 端口转发 `netsh interface portproxy` 实现。

## 三、启动流程优化（建议脚本）

新建 `/home/zlh123/start_lightrag.sh`：

```bash
#!/bin/bash
set -e

# 加载环境变量
if [ -f ~/.lightrag_env ]; then
    set -a
    source ~/.lightrag_env
    set +a
fi

# 动态获取 WSL 网关 IP
GATEWAY_IP=$(ip route show default | awk '{print $3}')
echo "→ WSL 网关: ${GATEWAY_IP}"

# 启动 LightRAG Server
exec lightrag-server \
  --llm-binding openai \
  --llm-model deepseek-chat \
  --embedding-binding ollama \
  --ollama-host "http://${GATEWAY_IP}:11434" \
  --embedding-model qwen2.5:1.5b
```

使用方式：`bash start_lightrag.sh`
